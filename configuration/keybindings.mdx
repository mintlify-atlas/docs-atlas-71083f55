## Global Keybindings

These keybindings are defined in `lua/fastfist/remap.lua` and are available globally across all buffers.

### Visual Mode Editing

<ParamField path="J" type="visual">
  Move selected lines down
  ```lua
  vim.keymap.set("v", "J", ":m '>+1<CR>gv=gv")
  ```
</ParamField>

<ParamField path="K" type="visual">
  Move selected lines up
  ```lua
  vim.keymap.set("v", "K", ":m '<-2<CR>gv=gv")
  ```
</ParamField>

<Tip>
These visual mode bindings allow you to move entire selections up and down while maintaining proper indentation.
</Tip>

### Navigation & Scrolling

| Keybinding | Mode | Description | Source |
|------------|------|-------------|--------|
| `<C-d>` | Normal | Scroll down half-page and center cursor | remap.lua:7 |
| `<C-u>` | Normal | Scroll up half-page and center cursor | remap.lua:8 |
| `n` | Normal | Next search result (centered) | remap.lua:9 |
| `N` | Normal | Previous search result (centered) | remap.lua:10 |

```lua
vim.keymap.set("n", "<C-d>", "<C-d>zz")

vim.keymap.set("n", "n", "nzzzv")
vim.keymap.set("n", "N", "Nzzzv")
```

  </Card>

### Completion (nvim-cmp)

Defined in insert mode during completion:

| Keybinding | Description |
|------------|-------------|
| `<C-e>` | Scroll docs up |
| `<C-y>` | Scroll docs down |
| `<C-p>` | Select previous completion item |
| `<C-n>` | Select next completion item |
| `<CR>` | Confirm selection |
| `<C-Space>` | Trigger completion |

```lua
cmp.setup({
    mapping = cmp.mapping.preset.insert({
        ['<C-e>'] = cmp.mapping.scroll_docs(-4),
        ['<C-y>'] = cmp.mapping.scroll_docs(4),
        ['<C-p>'] = cmp.mapping.select_prev_item(cmp_select),
        ['<C-n>'] = cmp.mapping.select_next_item(cmp_select),
        ['<CR>'] = cmp.mapping.confirm({ select = true }),
        ['<C-Space>'] = cmp.mapping.complete(),
    })
})
```

| Keybinding | Description |

## Quick Reference

<AccordionGroup>
  <Accordion title="Most Used Keybindings">
    - `<C-p>` - Find files
    - `<C-/>` - Search text
    - `<leader>n` - Toggle file explorer
    - `<leader>gs` - Git status
    - `gd` - Go to definition
    - `<leader>ca` - Code actions
    - `<leader>f` - Format code
  </Accordion>
  
  <Accordion title="Leader Key Commands">
    All commands starting with `<leader>` require pressing Space first:
    - Space → `n` for file explorer
    - Space → `gs` for git status
    - Space → `ca` for code actions
    - Space → `f` for formatting
  </Accordion>
</AccordionGroup>

<Tip>
You can view the leader key by running `:echo mapleader` in Neovim. It's set to Space in this configuration.
</Tip>
