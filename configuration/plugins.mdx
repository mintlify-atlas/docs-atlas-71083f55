This configuration uses [Packer.nvim](https://github.com/wbthomason/packer.nvim) as the plugin manager. All plugins are declared in `lua/fastfist/packer.lua` and configured in corresponding files under `after/plugin/`.

## Plugin Manager

<ParamField path="Packer.nvim" type="plugin">
  Plugin manager that can manage itself
  ```lua
  use 'wbthomason/packer.nvim'
  ```
  
  **Keybinding**: `<leader>s` runs `:PackerSync`
</ParamField>

<Info>
Run `:PackerSync` to install, update, and clean plugins. The command both installs missing plugins and removes unused ones.
</Info>

### Code Formatting

<ParamField path="Conform.nvim" type="formatter">
  Modern async formatter plugin
  
  `stevearc/conform.nvim`
  
  **Configured formatters**:
  - Lua: `stylua`
  - Python: `isort`, `black`
  - Rust: `rustfmt`
  - JavaScript/TypeScript: `prettierd` or `prettier`
  
  **Keybinding**: `<leader>f` to format current buffer
  
  ```lua
  use({
    "stevearc/conform.nvim",
    config = function()
      require("conform").setup({
          formatters_by_ft = {
              lua = { "stylua" },
              python = { "isort", "black" },
              rust = { "rustfmt", lsp_format = "fallback" },
              javascript = { "prettierd", "prettier", stop_after_first = true },
              typescript = { "prettierd", "prettier", stop_after_first = true },
          },
      })
    end,
  })
  ```
</ParamField>

## Plugin Loading Mechanism

<Steps>
  <Step title="Declaration">
    Plugins are declared in `lua/fastfist/packer.lua` using Packer's `use` syntax
  </Step>
  <Step title="Installation">
    Run `:PackerSync` to install/update plugins
  </Step>
  <Step title="Auto-configuration">
    Plugins with `config` functions are configured immediately (e.g., Comment, Conform)
  </Step>
  <Step title="After-plugin Configuration">
    Plugins without inline config are configured in `after/plugin/*.lua` files
  </Step>
</Steps>

<Info>
The `after/plugin/` convention means Neovim automatically sources these files after all plugins are loaded, ensuring proper initialization order.
</Info>

## Adding New Plugins

<Steps>
  <Step title="Add to packer.lua">
    Add a `use` statement in `lua/fastfist/packer.lua`:
    
    ```lua
    use 'author/plugin-name'
    ```
  </Step>
  <Step title="Install">
    Run `:PackerSync` in Neovim
  </Step>
  <Step title="Configure (if needed)">
    Create a new file in `after/plugin/` named `plugin-name.lua` with configuration
  </Step>
  <Step title="Add keybindings">
    Add any keybindings in your `after/plugin/` configuration file
  </Step>
</Steps>

<Tip>
For simple plugins that don't need configuration, you can add them to packer.lua without creating an after/plugin file.
</Tip>
